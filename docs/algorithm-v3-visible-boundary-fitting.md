# 算法 v3：可见边界曲线拟合 + 图层栈重建

> 状态：设计稿 v0.5（**主路径已落地**；取代 v2「圆/方/长条形状路由 + 色块贴章」）  
> 相关：[`algorithm-v2-curve-overlap-gpu.md`](./algorithm-v2-curve-overlap-gpu.md)（v2 历史；形状路由已废弃）  
> 补篇：[`algorithm-v3-shared-edge-and-planar-field.md`](./algorithm-v3-shared-edge-and-planar-field.md)（**设计中**：通用标签场概括 + 共享边缘拓扑，分批 A–E；治软边杂色与拼合细缝）  
> 约束：画布默认 320×320；图章库 256；层数 ≤40；变换含放大/旋转/平移/**出画布**；输出 `EmblemDocument` JSON + 渲染预览  
> CLI：`bfemblem approx` / `bfemblem prefit-stamps` / `bfemblem score`
---

## 0. 目标与非目标

### 0.1 要做什么

把任意输入图重建为 **有序、可重叠的图章层**，使：

1. **可见边界**（颜色相接处的轮廓）尽量贴合原图分割后的区域边界；  
2. 每层是库中真实图章经仿射（+ 可选镜像）后的结果，参数可导出到战地编辑器；  
3. 颜色以平面色为主，特殊图章可承担 **渐变/纹理感**；  
4. 在层数预算内 **自适应颜色粒度**：先粗后细，有空余层再加密。

### 0.2 明确不做

| 不做 | 原因 |
|------|------|
| **形状路由**（圆/方/长条桶强制候选章） | 把问题退化成几种模板贴图，违背「自动选取 + 边缘曲线拟合」 |
| 像素 Matching Pursuit 穷举小 ROI | 慢、且不懂「大章出画布只露一段弧」 |
| 场景特判（emoji/人脸/logo 白名单） | 不可扩展、难维护 |
| 追求照片级纹理 PSNR | 图章表达力不够；以剪影、色块、边界为准 |

### 0.3 核心心智模型

战地图章不是「把图切成拼图再选最像的贴纸」，而是：

> **前景盖住背景** 的有序色层。  
> 你在画布上看到的每一段轮廓，都是「某一层图章边缘」在未被更上层遮挡处的 **可见弧**。  
> 因此匹配对象是 **曲线（可见边界）**，不是整块 mask 的 IoU 唯一目标。

## 1. 问题形式化

### 1.1 图层

第 \(i\) 层（自底向上 \(i=1..n\), \(n\le 40\)）：

\[
L_i = (t_i,\; T_i,\; c_i,\; \alpha_i)
\]

| 符号 | 含义 |
|------|------|
| \(t_i\) | 图章 id（256 选 1，**由曲线匹配自动决定**，无人工形状桶） |
| \(T_i\) | 2D 相似/仿射：平移、旋转、均匀或非均匀缩放、flip |
| \(c_i\) | 染色 `fill`（平面色）；特殊章可保留多色纹理 |
| \(\alpha_i\) | 不透明度（默认 1；渐变/软边可用 &lt;1） |

可见形状：

\[
V_i = \mathrm{Canvas}\cap T_i(S_{t_i})\setminus \bigcup_{j>i} T_j(S_{t_j})
\]

最终颜色在像素 \(p\)：

\[
I(p) = \mathrm{over}\big(L_n,\ldots,L_1\big)(p)
\]

### 1.2 匹配的主对象：可见边界

对目标分割区域集合 \(\{R_k\}\) 与假设层序 \(\pi\)，每段 **可见边界** \(B\) 是：

- 某区域 \(R_k\) 的轮廓 \(\partial R_k\) 上的一段连续弧；  
- 且在该层序下，这段弧被解释为「形状 \(R_k\) 的外边界」而非被上层裁切出的假边（见 §3）。

对库中图章 \(t\)，取其边缘曲线（可多连通、含孔）\(\gamma_t\)。  
求 \(T^\star\) 使画布窗口内：

\[
T^\star = \arg\min_T \;
\mathcal{L}_{\mathrm{curve}}\big(T(\gamma_t)\cap\mathrm{Canvas},\; B\big)
+ \lambda_{\mathrm{soft}}\mathcal{L}_{\mathrm{soft}}
\]

其中 \(\mathcal{L}_{\mathrm{curve}}\) **允许有损**（对高频抖动、难弧可放宽），\(\mathcal{L}_{\mathrm{soft}}\) 为颜色/覆盖等软约束。

### 1.3 与「圆弧/椭圆弧基元」的关系

用户方案中的「轮廓分段 → 圆弧/椭圆弧逼近 → 再映射到图章」拆成两级：

| 级别 | 内容 | 作用 |
|------|------|------|
| **A. 解析基元层**（中间表示） | 每段用圆/椭圆/直线弧参数化 | 稳定、可优化、可解释层序 |
| **B. 图章实例层**（最终输出） | 每个基元或基元组由 **一个真实图章 + \(T\)** 实现 | 满足游戏只能放库内图章 |

并非每个圆弧都必须单独占一层：  
若某图章边缘本身含多段曲率，**一次 \(T\)** 可同时贴合多段目标弧（大章出画布时常见）。

## 3. 色彩平面化与区域、层序

### 3.1 平面化（P1）

- 色彩空间：CIELAB 聚类（或层次量化）。  
- **初始 K 偏大**（例如 8–12），以便边界清楚；不是一上来压成 3～4 色糊成一片。  
- 平滑：边缘保持（轻度），避免为「好贴章」过度抹掉结构。  
- 输出：标签图 \(L(x,y)\)、调色板 \(\{c_k\}\)、α。  
- **演进（补篇 Batch A/B）**：自适应重采样少造软边；平坦区估调色板；空间正则减杂色丝；**主体内 `gap_frac=0`**。详见补篇。

### 3.2 区域规整（P2）

- 每色连通域 → 区域 \(R_k=(mask_k, c_k, area, \partial R_k)\)。  
- 过小域并入邻域（阈值与画布比例相关，**与内容语义无关**）；**禁止**主体内标 -1 留洞。  
- 构建 **区域邻接图** \(G\)：边表示两色相接，权重为共享边界长度。  
- **演进（补篇 Batch C）**：邻接升级为 **SharedEdge 平面图**（几何只存一份），消灭「双侧独立轮廓」拼合缝。详见补篇。

### 3.3 层序假设 π（P3）——关键

平面化结果是 **并排色块**，游戏是 **堆叠色块**。必须把 2D 分割解释成深度：

**启发式（通用，非场景名）：**

1. **面积大、邻接多、更居中** 的区域更倾向底层；  
2. **被其它颜色包围** 的区域更倾向上层（洞/贴花）；  
3. **共享边界** 上，若一侧可被解释为「完整形状被另一侧盖住」，则完整侧更深；  
4. 可用能量：  

\[
E(\pi)=\sum_{\text{可见边 }e}\mathrm{cost}(e\mid\pi)
+ \mu\sum_k \mathrm{incompleteness}(R_k\mid\pi)
\]

- `cost`：该边作为「上层裁切」还是「形状固有边界」的合理性；  
- `incompleteness`：在 π 下，若某区域需由「大形状 + 上层挖空」解释，惩罚过碎挖空。

输出：有序区域列表（底→顶）及每条轮廓边的角色标签：

- `SHAPE_BOUNDARY`：应由图章边缘拟合；  
- `OCCLUSION_CUT`：由上层覆盖产生，**不**要求下层图章边缘经过此处。

这样「闭合色块相接」被正确读成「相接处至少有一侧是真边界」。

## 4. 轮廓分段与圆/椭圆弧逼近（P4）

### 4.1 提取

- 对每个 \(R_k\)：外轮廓 + 孔洞轮廓。  
- 多边形简化（RDP）仅作降噪，**最终匹配用较密采样点**。

### 4.2 分段

在曲率极值、转角超过阈值、或与邻接边角色变化处切开，得到弧段 \(\{B_m\}\)。

### 4.3 基元拟合

对每段 \(B_m\) 尝试：

| 模型 | 参数 | 适用 |
|------|------|------|
| 直线 | 点向式 | 平直边 |
| 圆弧 | 圆心、半径、角范围 | 常曲率 |
| 椭圆弧 | 中心、轴、旋转、角范围 | 透视/压扁圆 |
| 拒绝 | — | 拟合残差 > \(\varepsilon_{\mathrm{arc}}\) → 标为 `HARD_SEGMENT` |

**有损策略（用户要求）：**

- 连续变化不剧烈的曲线：允许 \(\varepsilon_{\mathrm{arc}}\) 放宽（例如相对弦长 2%～5%）；  
- `HARD_SEGMENT`：不强行单弧，改为 **多弧拼接** 或留给「复杂图章一次贴合多段」；  
- 评分时对 HARD 段降低 `line` 期望，避免无限加层硬抠。

中间表示示例：

```text
Primitive {
  type: line|circle_arc|ellipse_arc|free,
  params: ...,
  sample_points: (N,2),   # 画布坐标
  role: SHAPE_BOUNDARY|OCCLUSION_CUT,
  region_id: k,
  depth: π(k),
}
```

## 5. 图章边缘曲线库（P5）——自动选取的基础

### 5.1 构建（离线/首次缓存）

对每个图章 \(t=1..256\)（或用户子集，但 **默认全库可检索**）：

1. 高分辨率栅格 → α mask → 外轮廓/孔 \(\gamma_t\)；  
2. 弧长参数化重采样；  
3. 曲率序列、多尺度曲率直方图、可选 Fourier 描述子；  
4. 有向距离场 SDF 纹理（GPU）；  
5. **特殊效果标签**（自动，非场景）：  

| 标签 | 检测思路 |
|------|----------|
| `radial` | 从中心辐射的边缘密度 |
| `star` | 尖角周期、径向极值个数 |
| `grid` | 近似正交网格频率 |
| `ring` | 环形拓扑/孔 |
| `solid` | 高填充率单连通 |
| `gradient_candidate` | 灰度渐变或黑白条带 |

标签只用于 **渐变通道（P7）** 的候选加权，**不**用于普通色块的强制路由。

### 5.2 检索

对目标弧 \(B_m\)（或基元采样点）：

1. 描述子 top-M（如 M=48）从图章库取候选 \(t\)；  
2. **禁止**再映射到仅 4 个「圆章」——Circle 只是 256 中的一个候选；  
3. 对每个 \((t, B_m)\) 做几何对齐初始化（见 §6）。

## 6. 曲线匹配与变换搜索（P6）

### 6.1 变换空间

\[
T: \quad
\begin{aligned}
&\text{scale } s_x,s_y \in [s_{\min}, s_{\max}]\quad (s_{\max}\gg 1,\text{可数倍画布}),\\
&\text{angle }\theta \in [0,2\pi),\\
&\text{center }(c_x,c_y)\in [-C,W+C]\times[-C,H+C]\quad (C\sim W\text{ 允许出画布}),\\
&\text{flip } \in \{0,1\}^2.
\end{aligned}
\]

**整章 IoU 不得作为主损失**（出画布时失效）。主损失在曲线域：

### 6.2 曲线损失（有损）

对目标点集 \(P\subset B\) 与变换后图章边缘点 \(Q=T(\gamma_t)\cap\mathrm{Canvas}\)（或投影到可见部分）：

\[
\begin{aligned}
\mathcal{L}_{\mathrm{curve}}
&= \mathrm{Chamfer}(P,Q)
+ \beta\,\mathrm{NormalPenalty}(P,Q)\\
&+ \gamma\,\mathrm{CoverageGap}(P\to Q)
- \eta\,\mathrm{BonusLongArc}
\end{aligned}
\]

| 项 | 含义 |
|----|------|
| Chamfer | 双向点距；对 HARD 段可截断（Huber） |
| NormalPenalty | 切向/法向不一致 |
| CoverageGap | 目标弧未被盖住的比例 |
| BonusLongArc | 单次 \(T\) 连续贴合较长目标弧时奖励（鼓励大章） |

**软损失** \(\mathcal{L}_{\mathrm{soft}}\)：弧所属区域颜色与 `fill` 差；不必要的画布内多余边缘（鬼影边）。

### 6.3 优化（GPU）

1. **初始化**：  
   - 由圆/椭圆基元得 \(s,\theta,c\) 初值；或  
   - 弧端点 + 中点法向估计相似变换；  
2. **粒子群**（对数尺度 + 宽平移，**必须含出画布样本**）批量 `grid_sample` SDF/边缘；  
3. **局部精修**（Adam / 高斯牛顿于曲线残差）；  
4. 多假设保留 beam（每区域 top-b 个 \((t,T)\)）。

### 6.4 一层可服务多段

装配时优先选择「一个 \(T\) 同时降低多段 \(\mathcal{L}_{\mathrm{curve}}\)」的图章，减少层数、避免碎贴。

## 7. 特殊图章与渐变（P7）

平面色无法表达的平滑过渡：

1. 在 LAB 原图（平面化前）检测 **区域级渐变**：主方向、线性/径向；  
2. 若残差显著且层预算允许，从标签 `radial|star|grid|gradient_candidate` 中检索；  
3. 变换使图章调制方向对齐渐变方向；  
4. `opacity` 可 &lt; 1，或与底层平面色叠用；  
5. **验收**：不得引入强锯齿鬼边（`line_quality` 约束）。

这不是「为某 emoji 特判」，而是 **残差结构 → 效果类图章** 的通用通道。

---

## 8. 层栈装配与一致性（P8）

1. 按 π 从底到顶插入匹配成功的 \(L_i\)；  
2. 用真实/批量渲染器合成；  
3. 重提取预测图可见边界，与目标 `SHAPE_BOUNDARY` 比较；  
4. 失败处理：换 beam 次优章、分裂弧段、或把边降级为有损 HARD；  
5. 删除对最终可见贡献 &lt; ε 的层。

输出必须经过 **`EmblemRenderer`（与游戏一致）** 再评分。

---

## 9. 层预算与多级 K（P9）——用户「先多色、有空再加」

### 9.1 策略

```text
K = K_start          # 例如 8～12，偏多
loop:
  平面化(K) → 区域 → 层序 → 曲线拟合 → 装配 → n 层, Score
  if Score 达标 or n >= 40: break
  if n <= 40 - n_margin:   # 仍有空余层
     K = K + ΔK           # 增加颜色粒度
     或 split 高残差区域
     continue
  else:
     合并弱弧/弱区域，降低 K 或提高 ε_arc
     break
```

| 参数 | 建议起点 |
|------|----------|
| \(K_{\mathrm{start}}\) | 8～10 |
| \(\Delta K\) | +2 |
| \(K_{\max}\) | 14～16 |
| \(n_{\mathrm{margin}}\) | 4～8（留给渐变/细节） |
| 最大外环迭代 | 3～4（控制耗时） |

### 9.2 与「减少色块拼贴」的关系

- **多色** ≠ **多碎块**：规整与邻接合并仍要做；  
- 多色是为了 **边界更贴原图**，再靠大章曲线一次盖住，而不是每色抠碎章；  
- 空余层优先用于：未拟合 HARD 弧、渐变章、小面积上层贴花。

## 10. 评分（与输出）

### 10.1 分项

\[
\mathrm{Score} = S_{\mathrm{sim}}\cdot S_{\mathrm{line}}\cdot S_{\mathrm{simple}}
\]

| 分 | 内容 |
|----|------|
| \(S_{\mathrm{sim}}\) | 主体覆盖、平面色一致、粗结构 |
| \(S_{\mathrm{line}}\) | 可见边界规则度 + **与目标 SHAPE_BOUNDARY 的曲线误差**；锯齿/碎边 hard fail |
| \(S_{\mathrm{simple}}\) | 层数、有效层、避免无意义堆叠 |

曲线有损段不单独拖死 `sim`，但 `line` 对「乱抖边缘」仍严。

### 10.2 交付物

每次 `approx` 必须写出：

1. **预览 PNG**（`EmblemRenderer`）；  
2. **JSON**（`EmblemDocument`，含可出画布的 left/top/width/height）；  
3. **日志**：K 迭代、层数、stop_reason、分项分数、特殊章使用列表。

| 弧基元中间层 | `contour_arcs.py`（区域环 + **边上** `extract_primitives_for_shared_edges`） |
| 自适应重采样 + 标签场无洞 | `preprocess.detect_resample_mode` + `label_field.build_label_field`（`gap_frac=0`） |
| 共享边平面图 | `planar_map.PlanarMap` / `build_planar_map` / `planar_map_to_region_graph` |
| 装配缝宽 | `assemble.seam_p95_against_targets` → `ApproxMeta.seam_p95` |
| 特殊章渐变通道 | `special_fx.py`（radial/star/grid/gradient_candidate） |
| GPU 粒子服务曲线损失 | `torch_render.py` + `match_curve` 批量 `grid_sample` |
| 曲线+色彩综合评分 | `metrics.py`（可见边界 Chamfer × LAB 色 × line × simple，均 GPU） |
| SVG 预拟合 CLI | `bfemblem prefit-stamps` → `StampCurveLibrary.prefit_directory` |
| 图章曲线预拟合 | `stamp_curves.py`：默认 **256²** 栅格；**Moore 跟踪**外轮廓+内部孔；α≥96；多环描述子；缓存 v4 → `assets/.cache/stamp_curves` |
| 由简入繁 | `k_start=4` 粗阶段少色大块 + `coarse_max_regions`；细阶段升 K + 残差；特效仅细阶段 |
| 弧基元→图章 | 优质圆/椭圆基元时抬高区域简洁度、前置低 complexity 候选、完整形体播种 |

模块：

```text
approx/
  gpu_ops.py        # 共用 GPU 算子（形态学/CC/Chamfer/LAB/线质）
  planarize.py      # P1 多级 K（GPU k-means）
  regions.py        # P2 邻接图（GPU morph/CC/邻接）
  depth_order.py    # P3 层序 π
  contour_arcs.py   # P4 分段与圆/椭圆拟合（GPU SVD/lstsq）

  budget_loop.py    # P9

  torch_render.py   # 内环 GPU
```

## 12. 实现分期

### Phase A — 表示与数据（无路由）

- 删除/停用一切形状桶强制逻辑；  
- 区域邻接图 + 简单 π；  
- 轮廓分段 + 圆/椭圆拟合可视化（调试拼图：原图/标签/弧段/层序）。

### Phase B — 曲线匹配 MVP

- 图章全库（或 64+）曲线检索；  
- GPU 粒子 + Chamfer（有损 Huber）；  
- 出画布变换；  
- 输出 JSON+PNG；  
- 单测：合成「大圆+上层小圆」能否用出画布/遮挡解释。

### Phase C — 预算环与特效

- 多级 K；  
- 特殊章渐变通道；  
- beam 与遮挡一致性回退；  
- 性能：单张 GPU 目标 ≤15～30s。

### Phase D — 质量

- `line` 硬门槛与曲线误差联动；  
- 黄金集（几何、徽章、emoji、照片剪影）回归，**禁止**单样本特判。

## 13. 风险

| 风险 | 缓解 |
|------|------|
| 全库曲线匹配组合大 | 描述子 top-M + 弧段合并 + GPU 批量 |
| 层序 π 错导致边解释反了 | 多假设 π beam；装配后一致性检查 |
| 椭圆/圆弧不够表达复杂轮廓 | HARD 段 + 复杂图章一次多段；有损 |
| 渐变章引入脏边 | line_quality 否决；opacity 限制 |
| 与游戏渲染差 | 终局仅信任 EmblemRenderer |

---

## 14. 一句话

> **先把图看成「叠在一起的平面色形状」，再把每条真边界看成「某枚图章边缘在画布上露出来的一段曲线」；  
> 用放大、旋转、移出画布去对齐这些曲线（允许适度有损）；  
> 颜色先细后调层；空余层再加密或加特效章；  
> 自动选章只认曲线像不像，不认「这是不是圆/方/长条桶」。**
