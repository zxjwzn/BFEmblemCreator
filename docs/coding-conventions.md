# 编码规范

## 注解语言

**本项目所有注解类文本一律使用中文**，包括但不限于：

| 类别 | 要求 |
|------|------|
| 模块 / 包 docstring | 中文 |
| 类 / 函数 / 方法 docstring | 中文 |
| 行内 / 块注释 `# ...` | 中文 |
| Pydantic `Field(description=...)` | 中文 |
| CLI `help=` / Typer 说明 | 中文 |
| 面向用户的报错与 `typer.echo` 文案 | 中文 |
| 测试中的说明性 docstring / 注释 | 中文 |
| `README.md`、`docs/` 下项目文档 | 中文 |

### 允许保留英文的情况

- **标识符**：模块名、类名、函数名、变量名、JSON 字段名（如 `flipX`、`asset`）与编辑器/生态对齐，保持英文
- **专有名词与缩写**：SVG、RGBA、JSON、CLI、GPU、Pydantic、PIL 等可直接写
- **代码字面量**：路径、格式名（`PNG`）、第三方 API 参数名
- **与外部格式一一对应的键名**：编辑器导出 JSON 的字段名不得翻译
- **第三方库自身的版本称呼**（如「Pydantic v2」）可保留；**本仓库自研协议/缓存/文档不得自编号版本**

### 风格建议

- docstring 用完整句子，说明「做什么 / 约定是什么」，避免空话
- 注释写「为什么」或非显而易见的约束，不复述代码字面意思
- 用户可见错误信息应可据此排查（含关键值，注意不要过长）

## 实现与文档的时效性（无版本分叉）

1. **代码只有一套当前实现**，不维护并行旧路径，不写 `CACHE_VERSION` / `schema_version` / `v5`/`v6` 等多版本分支或迁移读法。
2. **缓存策略**：
   - 默认：对应缓存文件存在且字段完整则加载；缺失则只计算缺失项并写回。
   - **全量重算**仅由用户主动触发（如 CLI `prefit-stamps` / `force_refit=True`）。
   - 禁止用版本号或「改了代码就废全库」的隐式失效；算法变更后由用户主动预计算覆盖缓存。
3. **文档只描述当前行为**；已废弃方案若需对照，集中标为 **「旧设计」**，正文禁止「v2/v3/MVP/演进阶段/缓存 vN」叙事与修订版本表。
4. 注释写清「做什么/约束」，不写「相对于上一版改了什么」。
5. **禁止兼容旧功能的代码**：字段缺省回填、旧格式分支、`if key in files else 默认值` 等「读旧写新」一律删除；不完整缓存视为不存在并重算。

## 其它约定（摘要）

- 数据结构优先 **Pydantic v2** 建模；近似配置与中间结果字段应带中文 `Field(description=...)`
- 依赖与虚拟环境用 **uv** 管理
- 提交前通过：`ruff check`、`ruff format --check`、`pyright`、`pytest`
- 新增代码须符合本文件；发现英文注解应随改随译
