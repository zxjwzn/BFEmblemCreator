# 编码规范

## 注解语言

**本项目所有注解类文本一律使用中文**，包括但不限于：

| 类别 | 要求 |
|------|------|
| 模块 / 包 docstring | 中文 |
| 类 / 函数 / 方法 docstring | 中文 |
| 行内 / 块注释 `# ...` | 中文 |
| Pydantic `Field(description=...)` | 中文 |
| CLI `help=` / Typer 说明 | 中文 |
| 面向用户的报错与 `typer.echo` 文案 | 中文 |
| 测试中的说明性 docstring / 注释 | 中文 |
| `README.md`、`docs/` 下项目文档 | 中文 |

### 允许保留英文的情况

- **标识符**：模块名、类名、函数名、变量名、JSON 字段名（如 `flipX`、`asset`）与编辑器/生态对齐，保持英文
- **专有名词与缩写**：SVG、RGBA、JSON、CLI、GPU、Pydantic、PIL 等可直接写
- **代码字面量**：路径、格式名（`PNG`）、第三方 API 参数名
- **与外部格式一一对应的键名**：编辑器导出 JSON 的字段名不得翻译

### 风格建议

- docstring 用完整句子，说明「做什么 / 约定是什么」，避免空话
- 注释写「为什么」或非显而易见的约束，不复述代码字面意思
- 用户可见错误信息应可据此排查（含关键值，注意不要过长）

## 其它约定（摘要）

- 数据结构优先 **Pydantic v2** 建模
- 依赖与虚拟环境用 **uv** 管理
- 提交前通过：`ruff check`、`ruff format --check`、`pyright`、`pytest`
- 新增代码须符合本文件；发现英文注解应随改随译
